# v0.3.4 Centered Recommendation Modal Self Check

Date: 2026-07-06

## Version

- Product version: `v0.3.4`
- Codename: `居中推荐弹窗`
- Package version: `0.3.4`

## Problem

The recommendation card in v0.3.3 was still constrained by the fusion canvas. This made the generated recipe feel like an object inside the composition area rather than a page-level result.

## Change

- Added `.fusionResultLayer` as a fixed page-level overlay.
- Changed `.fusionResultDock` to a centered modal container.
- Kept card width constrained to the mobile app frame.
- Added a close button so users can dismiss the result and keep selected ingredients.
- Kept the existing card flip tutorial interaction.

## Browser Validation

Viewport: `390 x 844`

After fusion:

- `cardOffsetFromViewportCenter = { x: 0, y: 0 }`
- `cardOffsetFromCanvasCenter = { x: 0, y: 123 }`
- `cardInsideViewport = true`
- `hasLayer = true`
- `hasDismiss = true`
- `hasMeta = true`
- `pageCanScroll = false`
- `screenCanScroll = false`
- `documentElement.scrollWidth = 390`

Interaction:

- Tap `制作`: `flipped = true`
- Tutorial side has `制作教程` and `加入今天`
- Tap close: overlay disappears and selected ingredient remains selected.
- Re-fuse, tap `制作`, tap `加入今天`: navigates to Diary and shopping-list output is present.

## Product Judgment

The canvas is for composition. The recommendation is an outcome. Treating the result card as a centered page-level moment better matches the emotional purpose of the fusion interaction.

## Remaining Quality Gates

- Decide whether tapping outside the card should dismiss the result.
- Add recommendation explanation without crowding the modal.
- Persist generated recipe and selected workbench state after refresh.
