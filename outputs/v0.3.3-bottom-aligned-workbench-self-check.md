# v0.3.3 Bottom-Aligned Workbench Self Check

Date: 2026-07-06

## Version

- Product version: `v0.3.3`
- Codename: `贴底工作台`
- Package version: `0.3.3`

## Problem

The Warehouse workbench was still behaving like a scrollable page. The material tray used sticky positioning and inherited ordinary screen padding, so it felt fixed on top of the page instead of being part of a controlled mobile tool surface. It also left an unnatural gap from the bottom navigation.

## Change

- Added `.warehouseScreen` to isolate Warehouse from the normal page layout.
- Removed Warehouse screen padding and page-level scrolling.
- Changed `.fusionStudio` into a full-height grid.
- Set the upper fusion canvas to two thirds of the available middle screen area.
- Set the material tray to the bottom third and removed sticky positioning.
- Reduced tray item image/card/button heights.
- Kept horizontal rails inside the tray, but removed vertical page scrolling from the workbench.

## Product Judgment

The Warehouse tab should behave like a mobile creation workbench, not a content feed. A fixed two-zone layout makes the mental model clearer:

- Canvas: compose, preview, and receive the recipe result.
- Tray: choose ingredients, tools, preferences, and trigger fusion.

## Remaining Quality Gates

- Add an explainable recommendation layer without crowding the canvas.
- Decide whether the material tray should support collapse/expand later.
- Persist generated recipes and selected workbench state.
