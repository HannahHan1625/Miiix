# v0.3.6 Recipe Lifecycle Self Check

Date: 2026-07-10

## Scope

- Fix the awkward flow from fused recipe to favorite and cooking.
- Separate recipe candidate, saved recipe, meal plan, and cooked diary record.
- Make diary creation happen only after the user completes cooking.

## Implementation

- Bumped app version to `v0.3.6`.
- Added `MealPlan`.
- Added `savedRecipes` so generated fusion recipes can enter the recipe catalog.
- Changed `toggleFavorite` to accept a full `Recipe` object.
- Changed `planToday` to create a meal plan and shopping list without writing diary/cooked state.
- Added `completeCooking` to write the cooked diary record and cooked recipe state.
- Changed fusion recommendation actions:
  - primary: `今天做`
  - secondary: `收藏`
  - secondary: `看做法`
  - secondary: `再融合`
- Added a Today Plan panel in Diary with `开始制作` and `完成制作`.

- Manual check should cover: Warehouse -> select ingredients -> fuse -> favorite generated recipe -> Recipes favorite filter -> today plan -> start cooking -> complete cooking -> Diary record.

## Product Judgment

The lifecycle correction is more important than visual polish. The product should not write a diary record when the user only planned a dish. A generated dish must pass through clear states: candidate -> saved or planned -> cooking -> completed record.
