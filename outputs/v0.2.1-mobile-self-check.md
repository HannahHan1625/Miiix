# Miiix v0.2.1 Mobile Self Check

Date: 2026-06-30
Codename: 掌心仓库

## Result

This version fixes the main product-shell problem: Miiix is now rendered as a mobile-first app, not a wide desktop page.

Verified viewport: 390 x 844

- App frame width: 390px
- Document width: 390px
- Horizontal overflow: false
- Console errors: none
- Version shown in app: v0.2.1 掌心仓库

## Requirement Check

1. Navigation
   - Status: done
   - Evidence: bottom navigation has 4 items: 主页、仓库、菜谱、日记.

2. Home information summary
   - Status: done
   - Evidence: home includes username, persona badge, level score, and metric cards for inventory, cooking frequency, cooked dishes, and freshness.

3. Unified food upload entry
   - Status: done
   - Evidence: home has one upload entry. Upload sheet includes photo, online screenshot, receipt, and manual input methods. Completion closes the sheet and routes to Warehouse.

4. Ingredient information library
   - Status: done for MVP data model and UI
   - Evidence: food records include category, storage zone, storage tags, shelf life, amount/count mode, default quantity, price, calories, note, and freshness calculation.

5. Warehouse visualization
   - Status: corrected by latest product decision
   - Evidence: Warehouse no longer uses fridge styling. It directly shows ingredient summaries and grouped ingredient cards by storage zone. The fridge metaphor is deferred to a future home-personalization layer if needed.

6. Visual style
   - Status: partially done
   - Evidence: low-saturation blue/yellow palette, thin borders, small radius, and clean mobile cards are applied.
   - Quality gate: replace temporary remote food/tool photos with licensed internal cutout assets.

7. Recipes
   - Status: done
   - Evidence: recipes include filters for all/favorites/cooked and cuisine. Recipe cards show required ingredients, time, difficulty, cuisine, and calories.

8. Today Eat What flow
   - Status: done
   - Evidence: Warehouse and Recipes can add a dish to today's plan, produce a shopping list after subtracting owned ingredients, and add a diary record.

9. Kitchen tools
   - Status: done for interaction, partial for asset quality
   - Evidence: tools are horizontally scrollable and represented by retro product images.
   - Quality gate: replace temporary remote tool photos with a curated cutout asset library.

## Mobile Acceptance

- Home: pass
- Upload sheet: pass
- Upload completion to Warehouse: pass
- Warehouse ingredient direct display: pass
- Recipes: pass
- Diary: pass

## Strict Product Note

The next iteration should not add new concepts until the mobile MVP is visually trustworthy. The highest-leverage work is a real asset pipeline for food and kitchen-tool cutouts, plus persistent local storage.
